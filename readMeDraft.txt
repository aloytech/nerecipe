описание в виде ReadMe продолжу готовить...

Приложение не автономно - все храниться в FireBase
Таблица для категорий есть, но по факту, т.к. они константны используется внутренний  список.

Авторизация (выбор текущего пользователя) не реализована, "зашита" в пустую функцию getCurrentUser

На главном экране два фильтра по избранному и по категории работают совместно (и раздельно).
Поиск  ищет по всем рецептам, независимо от фильтра.
При активном фильтре по избранному, кнопка добавления в избранное работает некорректно.

Добавление\изменение картинок реализовано только для основной картинки рецепта. Не реализовано для этапов приготовления.

Свайп и перемещение реализовано только на экране редактирования рецепта (именно для редактирования). Соответственно этапы приготовления меняются местами, а по свайпу вправо происходит удаление, по свайпу вправо - переход к редактированию (при этом фактически этап удаляется из списка и добавляется в конец при сохранении)

Проверка корректности ввода происходит на уровне фрагмента, нельзя нажать сохранить при пустых полях (пустое имя или нет этапов), поэтому проверка на уровне добавления в репозиторий не осуществляется. Остальные поля (категория, автор, картинка) имею инициализированные значения всегда.
